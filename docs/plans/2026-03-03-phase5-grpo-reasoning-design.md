# Phase 5: GRPO Reasoning Training — Design

**Goal:** Add reinforcement learning-based reasoning training using Group Relative Policy Optimization (GRPO) with thinking mode, live tool execution, and a lightweight outcome reward model (ORM), incorporating 2026 research patterns beyond DeepSeek-R1.

## Architecture

Modern GRPO pipeline: multi-sample generation with tool interleaving → composite reward (rule-based + ORM) → group-normalized advantages → clipped policy gradient with KL penalty. Builds on SFT checkpoint as starting policy.

### Generation Pipeline

For each prompt in a training batch, generate G=16 completions independently:

1. Encode prompt: `<bos><user_start>question<user_end><assistant_start>`
2. Generate up to `max_gen_tokens=512` tokens per completion
3. During generation, if model emits `<tool_call_start>`:
   - Collect tokens until `<tool_call_end>`
   - Execute expression via sandboxed AST evaluator (picochat-tool)
   - Inject `<tool_result_start>result<tool_result_end>` into the sequence
   - Continue generating from there
   - Max 3 tool calls per completion
4. Model can generate `<think_start>...reasoning...<think_end>` blocks freely before the answer
5. Generation stops at `<assistant_end>` or max tokens
6. Each completion stores: token IDs, per-token log-probabilities

The generation function returns log-probs for every generated token (required for policy ratio computation). Tool result tokens are injected into context but excluded from the policy gradient since they weren't generated by the model.

### Reward Functions

Five signals combined into a composite score:

| Reward | Weight | Logic |
|--------|--------|-------|
| Accuracy | 1.0 | Binary 0/1. Extract final answer (strip think blocks, find `####` for math or letter for MC), compare to ground truth. |
| Format | 0.2 | 1.0 if `<think_start>`/`<think_end>` properly paired and before final answer; 0.0 if malformed or missing. |
| Tool use | 0.3 | 1.0 for correct invocation producing useful result; 0.5 for correct syntax but wrong/unused result; 0.0 for no/malformed tool use. Only applies when prompt has tool-use opportunity. |
| Length penalty | -0.1 | `max(0, (num_tokens - target_len) / target_len)`. Penalizes exceeding target length proportional to difficulty. No bonus for brevity. |
| ORM score | 0.3 | Value head scalar quality estimate for the full trajectory. |

Final reward: `r = 1.0*accuracy + 0.2*format + 0.3*tool + (-0.1)*length + 0.3*orm`

Answer extraction: strip `<think_start>...<think_end>` blocks, then find `####` marker (math), A/B/C/D letter (multiple-choice), or final numeric/string result (tool-use).

### ORM (Value Head)

Single linear projection from last hidden state at `<assistant_end>` token to a scalar. Trained with MSE loss against rule-based reward total (accuracy + format + tool — excluding ORM's own score to avoid circularity). Updated every GRPO step alongside the policy.

### GRPO Training Algorithm

```
Inputs:
  - policy model π (from SFT checkpoint)
  - reference model π_ref (frozen copy of SFT checkpoint)
  - value head V (linear layer, randomly initialized)
  - prompt dataset D (GSM8K + ARC + tool-use mixture)

For each training step:
  1. Sample mini-batch of prompts from D
  2. For each prompt, generate G=16 completions from π (with tool execution)
  3. Compute composite reward r_i for each completion
  4. Normalize advantages within group:
       advantage_i = (r_i - mean(r)) / (std(r) + 1e-8)
  5. Compute log-prob ratio per token per completion:
       ratio_t = exp(log π(token_t) - log π_old(token_t))
  6. Clipped surrogate objective:
       L_clip = min(ratio * advantage, clip(ratio, 1-ε, 1+ε) * advantage)
  7. KL penalty against reference:
       L_kl = β * mean(log π(token) - log π_ref(token))
  8. Value head loss:
       L_value = MSE(V(completion), rule_based_reward)
  9. Total loss:
       L = -L_clip + L_kl + α * L_value
  10. Update π and V (not π_ref)
```

Hyperparameters: ε=0.2, β=0.04, α=0.5, LR=1e-6 with cosine decay.

Memory optimization: generate all 16 completions in inference mode (no grad), storing token IDs + log-probs. Compute rewards and advantages. Second forward pass over top-k completions (by advantage) with gradients enabled. Backprop through ~4-8 completions, not all 16.

Reference model: frozen copy loaded once, used only for KL computation (forward-only).

### Tool Execution Engine (`picochat-tool`)

Sandboxed AST evaluator for safe expression execution during GRPO generation.

Supported operations:
- Arithmetic: `+`, `-`, `*`, `/`, `%`, `**`
- Math functions: `sqrt`, `abs`, `sin`, `cos`, `log`, `ceil`, `floor`
- String: `len("str")`, `"str".count("x")`, `"str".upper()`, `"str".lower()`
- Comparison: `==`, `!=`, `<`, `>`, `<=`, `>=`

Safety: 100ms hard timeout, max recursion depth 10, no variable assignment, no file/network/system access, division by zero returns error string.

### Training Data

Three sources with weighted mixture sampling (reuses `DatasetMixture`):

1. **GSM8K** (weight=0.5) — ~7.5K math reasoning problems. Verification: numeric answer after `####`. 4-shot CoT prefix.
2. **ARC-Challenge** (weight=0.3) — ~2.5K science multiple-choice. Verification: letter match. 5-shot prefix.
3. **Tool-use scenarios** (weight=0.2) — synthetic arithmetic/string problems requiring calculator. Verification: exact answer match + tool-use bonus. 2-shot prefix showing think + tool_call usage.

### Module Layout

**New crate: `picochat-tool`**
- `src/ast.rs` — Tokenizer, parser, AST types
- `src/evaluator.rs` — AST evaluation with safety constraints
- `src/lib.rs` — Public API: `execute_tool(expr) -> ToolResult`

**New in `picochat-engine`**
- `src/generate_with_logprobs.rs` — Generation returning token IDs + log-probs, with tool interleaving

**New in `picochat-train`**
- `src/grpo.rs` — GRPO training loop
- `src/rewards.rs` — Composite reward computation
- `src/value_head.rs` — ORM linear layer + MSE training

**New in `picochat-data`**
- `src/arc.rs` — ARC dataset loader
- `src/tool_data.rs` — Tool-use scenario loader

**Modified**
- `picochat-engine/src/lib.rs` — export new module
- `picochat-train/src/lib.rs` — export grpo, rewards, value_head
- `picochat-data/src/lib.rs` — export arc, tool_data
- `picochat-cli/src/main.rs` — add `--grpo` flag with config options

### CLI Integration

```
picochat --grpo \
  --checkpoint-dir ./sft-checkpoint \
  --gsm8k-data ./gsm8k_train.jsonl \
  --arc-data ./arc_challenge.jsonl \
  --tool-data ./tool_scenarios.jsonl \
  --group-size 16 \
  --total-steps 1000 \
  --max-gen-tokens 512 \
  --save-dir ./grpo-checkpoint
```
